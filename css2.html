How I Built This Website

Part 4: CSS II

Today, I am continuing with my CSS class.

Fonts

In order to change the font of your text, you need to assign a font-family. Font-families include serif, sans-serif and monospace. You can also assign a specific font like this: font-family:Arial;. A different font-family should be assigned as a back-up; Not every browser will know each font so it needs something to fall on if it cannot load the original value: font-family:Arial,sans-serif;. If your font name is more than one word, i.e. Times New Roman, you have to use quotation marks around it: font-family:"Times New Roman",serif;.

Most fonts won’t be known by your browser, so you have to download them into your file. The easiest way to do this is through Google Fonts, which offers a variety of fonts for free. It will also show you which font-family to assign as a back-up font. Previously, I decided to use the fonts Lato and Raleway for my website and downloaded them for each individual site into the <head> element like this:
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Lato:wght@100;300&family=Raleway:wght@300&display=swap">

I now learned that you can simply use the @import rule to load fonts into your CSS stylesheet, and then each site that is linked to said stylesheet can automatically use these fonts. I imported the fonts to my stylesheet like this:
@import url('https://fonts.googleapis.com/css2?family=Lato:wght@100;300;700&family=Raleway:wght@300&display=swap');
Google Fonts will automatically give you this code to embed into your file, all you need to do is simply copy and paste it into either your <head> element or your stylesheet.

You can change the size of your text by assigning a font-size. Font-sizes can be assigned in px;, which is a hard-set and doesn’t change even on responsive sites, or in rem;/em; which will be relative to the viewport and element sizes. You can also assign a font-style like italic; or normal;.

Text-Formatting

I was not new to the text-align property that I have used to center; my text. I did however not know about the letter-spacing property which spreads out letters and the word-spacing property which spreads out words. text-transform will turn your text into i.e. all lowercase; despite text syntax. So, for example <p style="text-transform; lowercase;">THIS IS ALL CAPS</p> will still show up in all lower case.

(While writing the previous paragraph, I learned about the <span> element, which allows to inline-style a few lines/words rather than an entire section)

You can also use inline formatting like <b> bold, <i> italic, <em> emphasized, <strong> strong, <del> delete or <mark> highlight.

Images

I was excited to learn more about images as I have been struggling with them a bit. I especially enjoyed learning how to create round corners and even turn the image in the ‘About Me’ section into a thumbnail-like circle by using the border-radius property. By cropping my photograph into a square and adding border-radius:50%;, my photo turned into a circle. I also added border:2px solid darkgrey; and padding:2px;. Check it out here.

(While writing the previous paragraph, I learned how to create an anchor element that links to a specific part of a website, rather than just linking to the website itself. I wrapped the 'About Me' section in an anchor element, assigning an id to it: <a id="about_me">SECTION<a>. You can then link to it as such: <a href="WEBPAGE#about_me>.)

Furthermore, I learned about the opacity property. It can have a value between 0 and 1. 0 means the element will be transparent, 1 means it will be opaque. I already defined opacity for .text-block in my <header>, but I used a different method. Rather than assigning the background-color through a rgb-value rgb(0,0,0), I used the rgba-value rgba(0,0,0,.8). The a defines the level of opacity.

To center an image, set the display property to block; and set a margin of auto;. If you want a responsive image, either the width or, preferably, max-width has to be set to 100%; with a height of auto;.

Padding & Margin

I had already used padding and margin, but wasn’t quite sure of their difference. margin creates space around/outside of the element, while padding creates space within the element. For both margin and padding, you can either be specific and assign a value to margin-left or padding-right, or you simply assign a value to margin or padding which will give the same value to -left, -right, -top and -bottom. Another way of specifically assigning values to -left, -right, -top and -bottom is this: padding: 20px 10px; In this scenario, 20px will be assigned to padding-top and padding-bottom, and 10px will be assigned to padding-left and padding-right. margin: 20px 10px 30px; will assign 20px to margin-top, 10px to margin-right and margin-left and 30px to margin-bottom. padding: 20px 10px 30px 40px; will assign 20px to padding-top, 10px to padding-right, 30px to padding-bottom and 40px to padding-left.

Grid System

I was not aware of this system. The idea is to create rows and columns that will allow you to easily position elements on the website. The grid system also works especially well for responsive websites as each column is given a certain percentage of the viewport and therefore easily shrinks and stretches out. All columns .col need to add up to 12 in order to take up exactly 100% of the viewport. The box-sizing property includes padding into the total width/height of the element. So, if it is set to border-box; it stops elements from getting too big and leaving the viewport. The clear property stops elements from floating after another element. Setting it to both; stops elements from floating on both sides.

At the moment, my system operates similarly. I have given <article> the width property of 65%; and <aside> the width property of 35%; so they are positioned next to each other and take up the entire viewport. I also placed both <article> and <aside> in a <section> together. <article> and <aside> represent columns in this grid scenario, <section> represents a row. After each <section>, I also set clear to both; and my box-sizing property is set to border-box; for all elements, as well.

GitHub Desktop

Unrelated to the class I’m taking, I finally learned how to use GitHub Desktop. Previously, I was creating and editing files in Brackets, and then manually copied each individual file and pasted it into the remote version of my website on github. This was highly impractical as it took a lot of time to copy and paste different files, and whenever I made a change to any file I had to go back and manually fix it either on github or on the local version in Brackets. Now that I am using the desktop version, I still create and edit files in Brackets, but Github Desktop automatically notices any changes I make to the local website, highlights them for me so I can check them one more time, and I can easily commit the changes to my master branch – no copy or paste needed. It seems silly now that I thought making manual changes to each individual file on both Brackets and github was the easiest way to work on my website.

What else?

I’ve been playing around with the :hover selector. At the moment, all links/anchor elements article a will be styled color:darkgrey; and article a:hover will be styled color:black;, whereas all links/anchor elements aside a will be styled color:black; and aside a:hover will be styled color:darkgrey;. This might change in the future, but it looks much better than the default setting. I also created a class .responsiveimg with the properties: max-width:100%; height:auto;. I now don’t have to set these properties individually each time I upload a new image.
