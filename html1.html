How I Built This Website

Part 1: HTML

I am currently taking a beginner’s class on full-stack web development over on Udemy. I have always been interested in how website’s work, and now that I have started my own blog, I feel like it is the perfect timing to finally learn this myself. My end goal is to fully build and host this website on my own, and I am especially interested in automation. At this moment, design-wise I am orienting myself on my “backup-blog” hosted on Wordpress, and I am mixing what I am learning in my class with my own research on the internet. I am a long way off from a fully-functioning and aesthetically pleasing website, but here you will get step by step descriptions of how I am building my first website, starting with the skeleton: HTML.

Navigation Bar

Underneath the header, I want a navigation bar that links to the different pages/categories of my blog. I could have used an unordered list (, bullet points) or ordered list (, numbered). In this case, each point would have to be contained in the list item tag.

However, in my Udemy class I was taught a trick that can make this look closer to an actual navigation bar rather than a navigation list: tables.

First of all, I am separating the head with an horizontal line using the tag. I then create a table using the tag. This table will have only one table row with each page having its own column. Each page/column is contained in the table data tag. The table will also have no border. With a bit of CSS I can set the border to 0 and play around with cellpadding and cellspacing until it almost looks like a proper navigation bar.

Pages

Now that Home or is essentially done, I have to create new pages. I could simply use copy & paste to get the exact same look on every single page, but as I am trying to properly learn HTML, I decided to write every single page by hand to get used to this new language. Github

I was initially quite worried about moving everything to github. I knew how to create the site but believed it would be quite difficult to link to other pages and I had no idea how to add images either. Once I had opened my github repository, though, it turned out everything was incredibly easy. Within the code section of the project’s repository, I could create new files/sites and link to them by simply referring to their complete name. I could also simply upload my .html files as a whole to the repository, and images can be uploaded the same way. Within just a few minutes I had moved all the sites I had created in Brackets to github, and then the website was live!
