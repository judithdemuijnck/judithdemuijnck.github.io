Part 2: HTML II

Contact Forms

In my last post, I wondered about contact forms, especially about how and where the submitted input is processed. After research, I found out that I have to specify the action attribute within the <form> element.

<form action="/action.page.php" method="POST" target="_blank">

In the example above, the input is submitted to a .php page. Eventually, I want to learn PHP as well, however it is only recommended if you already have basic knowledge of HTML, CSS and Javascript. For now my action attribute will be linking to an external form-handler (formspree.io). The method attribute defines how data is submitted. The default method is GET, which allows to bookmark results as the submitted input is shown in the URL. For sensitive data, POST should be used. The example also shows the target attribute which defines how the results will be opened. The default is _self, meaning it will open on the same page. With _blank results will open in a new window or tab. The target attribute works for other links as well.

I have also learned how to create a proper submit button. Rather than simply creating a button using the <button> element, I am now creating an input element: <input type="submit">. Furthermore, I realized each text input needs a name attribute, otherwise the data cannot be submitted.

W3schools is also using id, label and value attributes. To be honest, I haven’t quite figured out what these are for, but will include them in my final form just to be save.

For the message field, I am changing from <input type="text">, which would allow for only very little input, to <textarea>, which allows for multi-line input. I am also adding the required attribute to some elements. Furthermore, I found out about the <fieldset> element, which groups input together and creates a box around it, and the <legend> element, which is contained within the <fieldset> element and gives it a title. Meta Data

Once I finished my contact form, I moved on to adding meta data contained in the <head> element. I first declared the language of the website’s text using <html lang="en">. This is important so that screen readers can properly process the given information.

The <meta> element contains information for search engines, such as author information, descriptions and keywords: <meta name="author" content="Judith de Muijnck">. The charset attribute within the <meta> element defines the character encoding of the document. The most common value is UTF-8. Within the <meta> element you can also create a responsive website by controlling the viewport. Viewport is the visible area of a website, responsive means that the content will adapt to the screen size of the device used. So when on mobile phones, the viewport is much smaller, and the content needs to adapt to it.

<meta name="viewport" content="width=device-width, initial-scale=1">

Unfortunately this does not seem to work on tables, so my navigation bar is currently cropped on mobile. This is a problem I need to fix in the future.

Styling with CSS

Last time, I wondered how I could center text, and whether I needed CSS for this. While properly styling the website is only possible with CSS, it still seemed easy enough, so I eventually decided to add some styling already.

Using the <style> element, I centered my text with text-align:center. I also changed the background-color to a warmer color. Furthermore, I created a class .codesample that has a different background:color and font-family to highlight code in my posts. I then created a CSS stylesheet and loaded it into the HTML file. The great thing about stylesheets is that you can make changes to the styling and it automatically changes it on all the sites using the specific stylesheet.

<link rel="stylesheet" href="styles.css">

Layout

HTML allows you to structure your website using elements such as <header>, <nav>, <main>, <section>, <article>, <aside> or <footer. This makes it much easier to style certain sections. For example, I moved my blogposts into an <article> and styled it using text-align:center;, float:left; and width:65%;. I then created <aside> as a sidebar and styled it with float:left; and width:35%;. The sidebar will contain links to similar blog posts and information about myself.

Header

With my new (very basic) knowledge of CSS and some extra research, I updated my <header>. First of all, I cropped the photo I’m using (I did not use HTML or CSS to do this – but maybe this is possible as well) and then changed height to height:"auto"; and width to width=”100%”;. This made me realize that the percentage is referring to the amount of screen the image will take up, not actually the size of the photograph. I then moved my text on top of the picture by changing position to position:absolute; and rearranging the text top:50%;, left:50%; and transform:translate(-50%,-50%); so that it is centered on top of the image. By turning the background to background-color:rgba(0,0,0,.8); and updating the padding the text is now in a black, slightly transparent box. I am very happy with my header for now but am still looking forward to learning more about CSS and how to update it even more.

Navigation Bar

I also updated the navigation bar within the <nav> element. Previously, I created a <table> as my navigation bar as this automatically created some kind of padding between each cell. Now, however, I was able to delete <table> and rather changed padding to padding:16px;. I also changed the text to color:darkgrey; and then created a class .active to highlight active categories by changing to color:white;. I also added the :hover selector to style the text as color:white; when the mouse hovers over it. Give it a try!
