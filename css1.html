Part 3: CSS I

After already learning some CSS while tinkering with the styling of this website, I have now finally started my class on the topic. CSS stands for Cascading Style Sheet and is used to control and style designs and layouts of a website. There are three different ways to use CSS:

1. Inline
As the name suggests, CSS is added in line. Within the <body> element, you can chose any element and style it directly like this:
<h1 style="color:red;font-size:20px;">TEXT</h1>

Selectors

In the examples above, the elements have been directly targeted. It is also possible to create .classes or #ids to create stylings. .classes should be used if multiple elements are styled with same property and value. For example, I have created the class .codesample with the property background-color set to the value #EEEEEE;, the property font-family set to the value monospace; and the property fontz-size set to the value 10px;. I can use this class to distinguish and highlight code from regular text. #ids, on the other hand, should be used sparingly. They can have the same properties and values as .classes but are mainly used to interact with javascript.

Colors

Color values can be declared in three different ways:

  1. By a common name, such as color:red; or color:blue;
  2. With a hex code such as color:#fff; or color:#ffffff; (6 characters is more specific)
  3. RGB color such as color:rgb(0,0,0);

It is possible to include gradients and shadows as well.

Gradient Background with Shadows

For gradients, you set the background property to linear-gradient(direction, color1, color2);. More than two colors can be used. Top to bottom is the default setting but can be changed to any other direction. For shadows, you set the text-shadow property to value 1(horizontal shadow) value 2 (vertical shadow) value 3 (blur radius) color;. Blur radius and color are optional. All values can also be used for the box-shadow property.

Background Images

It is possible to use an image as your background by simply assigning a value to background-image. If the value (image) is too small, make sure to set background-repeat to no-repeat; to avoid a multiplied image. For an image to cover the entire viewport of the browser, use background-size:cover;. background-size is specific to the background-image property.

Borders

This text is surrounded by a dotted red border.

In this case, border is a summary for border-left, border-right, border-top and border-bottom. Each of these properties can be individually styled. For my website, I initially tried to either change the border property of <aside> to border-left:2px dotted darkgrey; or of <article> to border-right:2px dotted darkgrey; in order to have a soft separation between <article> and <aside>. However, I did not enjoy the way this looked as the dotted border did not actually separate anything when in mobile/responsive mode. Instead, I created a <div> within <aside> and assigned border:2px dotted darkgrey; to it. For now, I like the way this looks, especially in responsive mode.

Specificity

Each selector has a different weight value. So, when there are multiple declarations for the same element, there are certain rules as to which declaration is actually applied.

  • <elements> have a value of 1
  • .classes and [attributes] have a value of 10
  • #ids have a value of 100
  • Inline styling has a value of 1000

So, #ids override .classes, but inline styling (directly embedded in html) overrides all of the above. If there are multiple declarations and the weight value is the same (for example if there are two .classes styling the same element) then whatever was declared last "wins". !important overrides everything, no matter in what selector it is placed. It should be used very sparingly.
